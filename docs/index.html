SweetUrlHandler Documentation

The SweetUrlHandler class is a powerful tool for managing and handling URL routing and query parameters in PHP applications. It allows you to parse the request URI, match routes, and handle custom routes with ease.

Class Overview

The SweetUrlHandler class provides functionality to:

  • Parse the request URI and extract route segments and query parameters.
  • Match routes against patterns and execute callbacks.
  • Handle custom route patterns with user-defined handlers.

Constructor

The constructor initializes the route and query parameters based on the request URI.

public function __construct()

Methods

1. getRoute($index = null)

Gets the route segments as an array or a specific segment if an index is provided.

public function getRoute($index = null)

Parameters:

  • $index (int|null): The index of the route segment to return. If null, returns the entire route array.

Returns: Mixed - The route array or specific segment.

2. getParams($name = null)

Gets all query parameters or a specific parameter by name.

public function getParams($name = null)

Parameters:

  • $name (string|null): The name of the query parameter to return. If null, returns all parameters.

Returns: Mixed - The query parameters array or specific parameter value.

3. matchRoute($pattern)

Matches the current route against a given pattern and returns an associative array of parameters if matched, or false if no match.

public function matchRoute($pattern)

Parameters:

  • $pattern (string): The route pattern to match (e.g., 'products/{id}').

Returns: Array|false - Returns an associative array of parameters if matched, or false if no match.

4. handleRoute($pattern, $callback)

Matches the route against a pattern and calls a callback function if matched.

public function handleRoute($pattern, $callback)

Parameters:

  • $pattern (string): The route pattern to match.
  • $callback (callable): The callback function to execute if the route matches.

Returns: void

5. addCustomHandler($pattern, $handler)

Adds a custom route handler for specific patterns.

public function addCustomHandler($pattern, $handler)

Parameters:

  • $pattern (string): The route pattern to match.
  • $handler (callable): The handler function to execute if the route matches.

Returns: void

6. handleCustomRoutes()

Handles custom routes by checking all registered handlers.

public function handleCustomRoutes()

Returns: void

Example Usage

Here's a basic example of how to use the SweetUrlHandler class:

<?php
$handler = new SweetUrlHandler();

// Define a route and callback
$handler->handleRoute('products/{id}', function($params) {
    echo 'Product ID: ' . $params['id'];
});

// Add a custom handler
$handler->addCustomHandler('categories/{category}', function($params) {
    echo 'Category: ' . $params['category'];
});

// Handle custom routes
$handler->handleCustomRoutes();
?>

Conclusion

The SweetUrlHandler class is a flexible and powerful tool for managing URL routing in PHP applications. By leveraging its methods and functionality, you can efficiently handle routes and query parameters with minimal effort.
